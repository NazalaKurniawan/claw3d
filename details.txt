 CLAW3D — DETAIL PROGRAM LENGKAP
 3D AI Agent Visualization & Interaction Platform (v0.1.4)
================================================================================

[1] DESKRIPSI UMUM
──────────────────
Claw3D adalah aplikasi open-source Next.js untuk memvisualisasikan dan
berinteraksi dengan AI agents yang dijalankan oleh OpenClaw.

- Stack: Next.js 16 + React 19 + Three.js + TypeScript + TailwindCSS
- Bukan runtime AI — hanya layer visual & proxy di atas OpenClaw Gateway
- Menyediakan: workspace agents, 3D office, builder, studio proxy
- Lisensi: MIT

Visi: Membangun "kota 3D" di mana AI agents beroperasi sebagai entitas
visual yang bisa diobservasi, diinspeksi, dan diajak berinteraksi.

[2] ARSITEKTUR 4 BAGIAN
────────────────────────
┌─────────────────────────────────────────────────────────────┐
│  User  →  Browser UI (Next.js)                              │
│            ↓ HTTP          ↓ WebSocket                      │
│       Studio API Routes   Studio WS Proxy (/api/gateway/ws) │
│            ↓                    ↓                           │
│       Local Filesystem    OpenClaw Gateway (upstream)        │
└─────────────────────────────────────────────────────────────┘

1. Browser UI       → Next.js client: agents workspace, office, builder
2. Studio API Routes → Server routes: local settings, file ops
3. Studio WS Proxy   → Node server: proxy WS ke upstream OpenClaw Gateway
4. OpenClaw Gateway   → Source of truth: agents, sessions, config, runtime

[4] SERVER LAYER (server/)
──────────────────────────
  index.js                    → Entry point: boot Next.js + HTTP/HTTPS server
  gateway-proxy.js            → WebSocket proxy ke upstream OpenClaw Gateway
  studio-settings.js          → Load local Studio gateway settings
  access-gate.js              → Token-based access control
  network-policy.js           → Host validation & public host safety
  demo-gateway-adapter.js     → Demo mode (tanpa OpenClaw/Hermes)
  hermes-gateway-adapter.js   → Hermes Agent adapter (alternatif OpenClaw)

Server flow:
1. Load env & resolve hosts/port
2. Assert public host security (token required jika public)
3. Create Next.js app + HTTP/HTTPS server
4. Setup WebSocket proxy (/api/gateway/ws → upstream gateway)
5. Access gate: token verification untuk HTTP & WS

[7] 3D OFFICE (src/features/retro-office/)
──────────────────────────────────────────
Powered by: React Three Fiber + Three.js

  RetroOffice3D.tsx (273KB) → Main 3D scene renderer
    - Renders agents sebagai aktor 3D bergerak
    - Room-based spatial layout
    - Derived animation dari runtime events
    - Procedural furniture geometry (bukan 3D model bundled)

  core/
    navigation.ts      → Nav grid, pathfinding, route helpers
    furnitureDefaults.ts → Default room/object layout
    geometry.ts        → Object sizing, bounds, snapping
    navigation/
      gymRoute.ts         → Gym room routing
      serverRoomRoute.ts  → Server room routing
      qaLabRoute.ts       → QA lab routing

  objects/
    furniture.tsx   → Desk, chair, shelf renders
    primitives.tsx  → Walls, floors, basic shapes
    machines.tsx    → Server rack, monitor objects

  systems/         → Scene systems (lighting, camera, etc)
  overlays/        → 2D overlays on 3D scene
  scene/           → Scene setup & configuration

Motion System (derived, bukan imperative):
  1. Runtime events masuk dari gateway
  2. reduceOfficeAnimationTriggerEvent() → record latches
  3. reconcileOfficeAnimationTriggerState() → derive durable holds
  4. buildOfficeAnimationState() → shape untuk scene
  5. RetroOffice3D → render destinations, paths, overlays

[10] SKILLS SYSTEM
──────────────────
Skills = unit kemampuan reusable untuk agents (format AgentSkills.io)

Struktur skill:
  - SKILL.md (frontmatter: name, description, metadata)
  - Gating: bins, env, config, OS requirements
  - Eligibility: computed setiap load/snapshot

6 sumber skill (prioritas rendah → tinggi):
  1. Extra dirs / plugin skills
  2. Bundled skills
  3. Managed global (~/.openclaw/skills)
  4. Personal agents (~/.agents/skills)
  5. Project agents (<workspace>/.agents/skills)
  6. Workspace skills (<workspace>/skills)

Gateway API: skills.status, skills.install, skills.update, skills.bins
